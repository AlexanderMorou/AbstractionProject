Some notes about the Common Language Infrastructure Type System.

The CLI Type System imports Microsoft.VisualBasic to support checking for SpecialClassModifier feedback. Visual Basic.NET appends Microsoft.VisualBasic.CompilerServices.StandardModuleAttribute to classes marked with the Module keyword, as of .NET version 2.0. There are a few difference between Modules in Visual Basic.NET, and static classes in C♯:

  1. C♯ static classes are abstract and sealed, whereas Visual Basic.NET Modules are simply sealed.
  2. The static modifier on members is required in C♯, if omitted the C♯ compiler will throw an error; while the Shared modifier is optional in Visual Basic.NET Modules, the compiler infers it for you.
  3. Modules cannot be partial, the entire Module must be declared in one file. static classes do not have this restriction.
  4. In Visual Basic.NET Modules cannot be nested whereas, in C♯, static classes can be.
